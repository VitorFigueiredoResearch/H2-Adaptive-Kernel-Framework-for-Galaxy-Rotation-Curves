2. NATURE OF THE INCONSISTENCY (now resolved)
----------------------------------------------
The original diagnostic_report.txt contained these simultaneous statements:

  Galaxies successfully run:  74
  Galaxies excluded total:     6
  Galaxies with n_inner >= 3: 74
  Galaxies with n_inner <  3:  0
  LOW INNER-REGION POINT COUNT:  None

These appeared contradictory: if 6 were excluded and the only plausible
exclusion reason was n_inner < 3, how could "n_inner < 3 = 0"?

ROOT CAUSE:
  The diagnostic_report_template.py script counted n_inner from the
  mond_perturbation_summary.csv, which contains ONLY the 74 galaxies that
  passed the n_inner >= 3 gate. The 6 excluded galaxies were filtered out
  BEFORE any row was written to the CSV. The script therefore correctly
  found 0 galaxies with n_inner < 3 in the CSV, but misreported this as
  "0 galaxies excluded for this reason", omitting the pre-CSV filter stage.

RESOLUTION:
  The inconsistency was ONLY in the text report — the CSV data and the
  scientific results are correct and unaffected. The diagnostic_report.txt
  was rewritten to distinguish between:
    (a) "n_inner < 3 in CSV = 0" (correct, refers to CSV contents)
    (b) "n_inner < 3 causing exclusion = 6" (correct, refers to pre-CSV filter)
  The 6 excluded galaxies are now explicitly named with their inner-point counts.
